🧹 Refactor ExceptionManager.request_exception to use a single data class

🎯 **What:** The `request_exception` method in `ExceptionManager` accepted 5 distinct string parameters, making it overly complex. I refactored it to accept a single Pydantic model `ExceptionRequestInputRecord` encapsulating these parameters.

💡 **Why:** Encapsulating related parameters into a single object reduces function signature complexity, improves readability, and makes future parameter additions easier, significantly enhancing code health.

✅ **Verification:** I ran `flake8` and `black` to ensure formatting and linting consistency, and executed the conformance-related test suite, confirming zero broken tests and preserved functionality.

✨ **Result:** A cleaner and more maintainable `request_exception` method leveraging standard Pydantic models.
