# Operations Handbook Overview

Welcome to the Sports Signal Bot Operations Handbook. This is the central hub for operating, maintaining, and developing the forecasting system.

## Information Architecture
- **[Architecture](architecture/system_overview.md):** System overviews and component maps.
- **[Operators](operators/operator_handbook.md):** Guides for daily slot runs, review queues, and channel monitoring.
- **[Runbooks](runbooks/routine/morning_slot_live_like.md):** Step-by-step procedures for routine, maintenance, recovery, and release tasks.
- **[Incidents](incidents/stale_artifacts.md):** Playbooks for responding to specific failures and degraded states.
- **[Onboarding](onboarding/getting_started.md):** Guides for new operators and developers.
- **[Reference](reference/cli_reference.md):** CLI, configuration, and glossary resources.
- **[Governance](governance/release_governance.md):** Policies for releases, schema changes, and quality gates.

## Role-Based Entry Points
- **I am an Operator:** Start with the [Operator Handbook](operators/operator_handbook.md) and the [Daily Checklist](onboarding/day1_checklist.md).
- **I am responding to an Incident:** Go directly to the [Incident Playbooks](incidents/stale_artifacts.md).
- **I am doing a Release:** Read the [Release Governance](governance/release_governance.md) and [Canary Validation Runbook](runbooks/release/canary_validation.md).
