💡 **What:** Replaced the explicit `for` loop used to gather all fields with a one-liner `set().union(*(obs.payload for obs in group.observations))`.
🎯 **Why:** Creating an iterator via `.keys()` and updating the set in a loop is slow for many payloads. Using `set().union()` with a generator iterates directly over the dict keys (via the dicts themselves) and executes at C-level, avoiding extra Python bytecode operations.
📊 **Measured Improvement:** In a benchmark creating 50 observations with 20 keys each (tested over 50,000 iterations), the execution time improved from 2.17s to 1.70s (about a ~22% improvement over the baseline).
