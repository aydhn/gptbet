1. Phase 44 implementation summary
Added Batch Candidate Tournament architecture (Phase 44) to sports_signal_bot.
This phase elevates the system from evaluating single suggestions in isolation to evaluating multiple candidate patches concurrently using a shared evaluation universe.
- Created core contracts, strategies, and logic for candidate grouping, comparability validation, pareto front, dominance relation logic, constraint filtering, safety lane assignments, secondary ranking, recommendations, gate burden analysis, and evidence bundling.
- Integrated with main app through Typer CLI and added tests covering the new functionality.
- Generated sample configs and documentation files as specified.
